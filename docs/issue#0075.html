Design Decisions

Decision Qualify leaf symbols with agentcore. rather than dot-import

Ambiguity: The moved provider files referenced leaf types (ContentList, AssistantMessage, AgentTool…) by bare name, resolved through package agent. In package provider they need a source; the PRD doesn't say whether to dot-import agentcore or qualify.

Choice: Explicit agentcore. qualification on every moved-leaf reference; no dot-import.

Rationale: Dot-imports are discouraged in Go (obscure symbol origin, vet-unfriendly). Explicit qualification makes the agentcore dependency legible at every use site — exactly the layering the split is meant to make visible.

Decision Separate provider_shim.go from the existing agentcore_shim.go

Ambiguity: Package agent already carries agentcore_shim.go from step 1. The new provider re-exports could go there or in a new file.

Choice: A dedicated internal/agent/provider_shim.go, one shim file per extracted package.

Rationale: Each shim's lifetime is tied to its package's caller-migration step; keeping them separate makes each independently deletable and keeps the diff for each US self-contained.

Deviations

Deviation Shim surface is larger than the 7 symbols named in the AC

Spec: US-002 lists Provider, StreamFn, StreamFnFromProvider, Model, NewOpenRouterProvider, NewOllamaProvider, NewCredentialStore as the symbols to export "correctly".

Implemented: The shim re-exports the full moved surface (~40 symbols: all Stream*Event types, decoders, TransportConfig, ModelRegistry, thinking translators, auth loaders, etc.).

Why: The AC list is illustrative ("等" / "etc."), not exhaustive. Every exported symbol the remaining agent files reference must be aliased or package agent won't compile — the build itself dictates the complete set. Narrowing to 7 would break the build.

Tradeoffs

Tradeoff Build-driven shim enumeration vs manual audit

Alternatives: (a) enumerate the exported decls by reading each moved file, then add aliases; (b) move files, let go build surface every "undefined: X", add aliases reactively.

Chosen: a hybrid — read the files to seed the shim, then let the compiler prove completeness.

Why it won: The compiler is the ground truth for "what does package agent still need." Relying on it guarantees no missing alias and no over-exporting of symbols that aren't actually referenced.
