Issue #40 — [impl] 系统提示词 + AGENTS.md 注入（US-021）— 2026-07-10

Design Decisions

Decision AGENTS.md 遍历以显式 Root 为上界，而非无限向上到文件系统根

Ambiguity: 验收说"从根到子目录按通用→具体注入"，但"根"指仓库根还是文件系统根未定义。

Choice: PromptConfig.Root 显式界定遍历上界；从 WorkingDir 向上收集到 Root（含）再反转成 root-first。Root 为空时只看 WorkingDir 自身，不做祖先遍历。

Rationale: "根"= 仓库根，由调用方（cmd/pigo 传 cwd）决定，避免误读用户 HOME 或 / 下的无关 AGENTS.md；显式边界也让行为可测、可预期。

Decision 缺失的 AGENTS.md 静默跳过，仅真实 I/O 错误上报

Ambiguity: 验收未提"某层没有 AGENTS.md"或"文件不可读"该如何处理。

Choice: os.IsNotExist → 跳过并继续；其它错误（如权限）→ 包装后返回。空内容文件也跳过。

Rationale: monorepo 中大多数目录本就没有 AGENTS.md，缺失是常态而非错误；但"文件存在却读不了"是需暴露的真实故障。

Decision 注入 Now/ReadFile 依赖以支持确定性单测

Ambiguity: 验收要"单测验证注入顺序"，但顺序依赖真实磁盘布局与时钟，难以稳定断言。

Choice: PromptConfig 暴露可选 Now func() time.Time 与 ReadFile func(path)([]byte,error)，默认 time.Now/os.ReadFile。测试用 map 伪造 AGENTS.md 布局。

Rationale: 让顺序断言不触碰磁盘、不受当天日期影响，测试稳定且快速——与仓库既有"注入 seam"风格一致。

Tradeoffs

Tradeoff 向上收集再反转 vs 先算深度再自上而下

Alternatives: (A) 从 wd 用 filepath.Dir 逐级上行到 root，收集后反转成 root-first；(B) 用字符串前缀/分隔符计数直接自上而下构造路径。

Pros/Cons: B 需处理分隔符、卷标、.. 等平台差异，易错；A 复用 filepath.Dir/Clean 的既有语义，且天然处理"wd 不在 root 之下"（走到文件系统根仍未命中 → 回退 wd 独一）。

Winner: A——更少的手写路径运算，边界条件由标准库兜底。
