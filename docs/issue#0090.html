Implementation Notes

Issue #90 — TUI: 用 bubbles textinput 修复中文输入与 rune 级编辑 — 2026-07-10

Design Decisions

Decision textinput 归 Model 持有，uiState 保持 framework-free

Ambiguity: AC 只要求「用 textinput 替换 uiState.input 的手写拼接」，没规定 widget 放在纯状态层还是 bubbletea 桥接层。

Choice: 把 textinput.Model 放进 Model（model.go），从 uiState 删除 input string 字段；submit 改为接收 input string 参数。

Rationale: PRD 的 Design Considerations 明确要求「保持 uiState 无 bubbletea 依赖、可单测」（FR-14）。textinput 是 bubbletea 组件，若塞进 uiState 会污染纯状态层。让 model 拥有 widget、把值作为参数传给 submit，既修了中文输入又守住了分层。

Decision 除 Ctrl+C / Enter 外的所有按键统一委派给 textinput

Ambiguity: AC 要求「替换 handleKey 的 default/backspace 分支」，但没说要不要保留独立的 backspace case。

Choice: 删掉独立的 backspace case，与 default 合并为一个「委派给 m.input.Update(k)」分支；backspace、方向键、rune 插入全由 textinput 的 KeyMap 处理。

Rationale: textinput 的 Update 已内建 DeleteCharacterBackward（按 rune 删）、CharacterForward/Backward（按 rune 移光标）等绑定。自己再拦 backspace 反而要重新实现 rune 逻辑，与委派冲突。合并后一处委派覆盖所有编辑键，天然满足「rune 级移动 + 退格删整字」。

Decision View 保留自绘 "> " 前缀，textinput 的 Prompt 置空

Ambiguity: textinput 自带 Prompt（默认 "> "），而原 View 也手写 "> "。

Choice: ti.Prompt = ""，继续由 View() 输出 "> " 再拼 m.input.View()。

Rationale: 最小改动、不重复前缀；后续 #94/#95 统一改造 composer 渲染时再决定前缀归属，本 issue 不扩大范围。

Deviations

None 实现遵循 spec

AC 各项（textinput 替换手写拼接、中文完整显示不丢字、rune 级光标移动、退格删整个 rune、submit 文本逐字一致、保留空输入忽略/steering/Ctrl+C-disarm 语义、含中文用例的单测全绿）均按字面满足，无偏离。

Tradeoffs

Tradeoff submit 改为接收参数 vs. 让 uiState 读 widget

Alternatives: (a) submit(input string) 由 model 把 m.input.Value() 传入；(b) 在 uiState 里保留某种输入引用，让 submit 自己取值。

Chosen: (a)。

Why it won: (b) 会把 bubbletea 组件的引用漏进纯状态层，破坏可单测性。(a) 只多一个字符串参数，改动集中在 submit 签名与其调用点/测试，代价小且分层干净。

Tradeoff 新增 model_test.go 驱动 handleKey vs. 只测 uiState

Alternatives: (a) 只在 state_test.go 层测 submit；(b) 另建 model_test.go，用合成 KeyPressMsg（{Code:r, Text:string(r)}）直接驱动 handleKey。

Chosen: (b)（同时保留 a）。

Why it won: 中文输入的 bug 恰恰在 handleKey（旧 len(s)==1 分支丢多字节），只测 uiState 覆盖不到。model_test.go 用不依赖真实终端的合成按键，端到端验证「键入你好世界 → 完整显示 → 退格删整字 → 提交逐字一致」，正对 bug 根因。

Open Questions

Question 引入了 github.com/atotto/clipboard 间接依赖（textinput 的 Paste 绑定）

textinput 依赖 clipboard 实现粘贴，go mod tidy 已将其纳入 go.sum。本 issue 未主动使用粘贴功能，但按键委派后 Ctrl+V 会触发 textinput 的 Paste 命令。确认接受这一新间接依赖；粘贴行为的中文正确性可在实机确认。

Question 光标目前用 textinput 的真实终端光标

newComposer 未开 SetVirtualCursor，走真实光标路径。当前 View() 直接拼 m.input.View()，未处理 Cursor() 位置上报。在超宽输入折行/滚动（US-003/#91 宽度感知）接入前，长中文输入行的光标定位以实机为准复核。
