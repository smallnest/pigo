类图 (Class Diagram) — 结构性图形。展现 Provider 抽象层静态结构：接口与两种驱动实现、传输/解码器契约、事件密封接口、注册表与凭证解析。

统一流式接口 · 双驱动

Provider · openAICompatDriver · anthropicCompatDriver

  • Provider 是唯一抽象：Name() / Models() / StreamCompletion()。
  • 两种实现分别封装 OpenAI Chat Completions 与 Anthropic Messages 两套线协议。
  • 双失败模型 (FR-13)：建流阶段失败以 error 返回；运行期失败以终态 ErrorEvent 落在流上，绝不再返回 error。
  • StreamFnFromProvider 将 Provider 适配为函数式 StreamFn。

共享传输 · 有状态解码 · 注册凭证

TransportConfig · Decoder · ProviderSpec · CredentialStore

  • StreamRequest = connect + 后台 pump，双看门狗（idle 5m、stall ×1.2），429/503/529 仅在建连阶段重试。
  • Decoder 由 OpenAIDecoder / AnthropicDecoder 实现，逐 SSE payload 增量解码为 AssistantMessageEvent（6 种密封事件）。
  • 注册表 ProviderSpec（约 38 个内置，含千帆/火山/百炼/混元）只存 env 变量名。
  • CredentialStore.GetAPIKey：OAuth → override → env → config；密钥仅在请求时解析，从不记录。
