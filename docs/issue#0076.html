Design Decisions

Decision Export executeToolCalls as ExecuteToolCalls, alias lowercase in shim

Ambiguity: executeToolCalls was unexported in package agent and is still called by loop.go (which stays in agent). Once the function moves to agenttool, an unexported name is invisible across the package boundary; the AC lists the exported tool symbols but not this internal one.

Choice: Export it as agenttool.ExecuteToolCalls, then bind var executeToolCalls = agenttool.ExecuteToolCalls in the shim so loop.go compiles verbatim.

Rationale: Identical to the step-1 emitFunc → EmitFunc treatment — cross-package visibility forces the capitalization, and the lowercase alias keeps the caller untouched until US-004/005 migrates it. Consistency across shims makes the pattern predictable.

Decision Re-provide echoTool/execTool test helpers in a new testtools_test.go

Ambiguity: echoTool/execTool were defined in batch_executor_test.go/tool_executor_test.go (which move to agenttool) but are also used by loop_test.go, headless_test.go, orchestration_test.go, faux_provider_test.go that stay in package agent. Unexported test-only symbols cannot cross the package boundary.

Choice: Add internal/agent/testtools_test.go re-providing the two helpers for the agent-resident tests; the canonical copies move with their files to agenttool.

Rationale: Test scaffolding only — no production symbol touched. The alternative (exporting them) would leak test types into the production shim surface. Duplicating ~30 lines of test helper is cheaper and cleaner; both copies disappear when US-004 migrates the agent-resident tests to runtime.

Deviations

Deviation Shim surface exceeds the symbols named in the AC

Spec: US-003 names BashTool, EditTool, ReadTool, WriteTool, GrepTool, FindTool, ToolRegistry, NewToolRegistry, ToolExecutorConfig, BatchConfig as the exports.

Implemented: The shim additionally re-exports LsTool, FieldError, ValidationErrorResult, the sandbox surface (Decision, AccessKind, Request, Policy, SandboxGate, NewPolicy + Decision*/Access* consts), and the secret surface (SecretRegistry, NewSecretRegistry).

Why: Same build-driven rule as US-002: the AC list is illustrative, the compiler is ground truth. Every symbol the remaining agent files reference must be aliased or package agent won't build. The complete set is dictated by go build surfacing "undefined: X", not by the prose list.

Tradeoffs

Tradeoff Duplicate test helpers vs export them vs move dependent tests now

Alternatives: (a) re-provide echoTool/execTool as a small agent-side test file; (b) export them from agenttool and reference across packages; (c) migrate loop/headless/orchestration tests to runtime now instead of US-004.

Chosen: (a).

Why it won: (b) pollutes the production package with test-only types and can't work anyway (test symbols aren't in the non-test build). (c) balloons #76's scope into US-004's territory, breaking the one-layer-per-US invariant. (a) is the minimal, self-contained, trivially-deletable option.
