Design Decisions

Decision 路径以 Root 为界，用 filepath.Rel 做越界检查

Ambiguity: 验收要"路径越界明确报错"，未规定"越界"如何定义。

Choice: ReadTool.Root 界定可读目录；resolvePath 用 filepath.Join+filepath.Rel 判定，rel 为 .. 或以 ../ 开头即拒绝。

Rationale: 单一 Root 边界简单可测；Rel 在 Clean 后比较可挡住 ../ 穿越，测试用真实的 root 外 secret 文件验证。

Decision 读失败编码为 error result，Go error 仅留给参数解码

Ambiguity: 工具执行失败该返回 Go error 还是 result？

Choice: 文件不存在/越界/是目录等均返回 errorResult(...)（IsError result），只有 json.Unmarshal 失败才可能返回 Go error（此处也降级为 result）。

Rationale: 对齐既有 tool_executor 的 dual failure 语义——工具运行失败让模型看到并重试，而非中断 loop。

None 无偏离

实现覆盖验收：按路径读、offset/limit 分页、带行号输出、超大文件（>2000 行）截断提示、越界/不存在明确报错、t.TempDir 行为单测、go test ./... 通过。

Tradeoffs

Tradeoff 截断按行数（2000）而非字节数

Alternatives: (A) 按行数上限截断；(B) 按总字节数截断。

Pros/Cons: A 与"行号输出/offset 分页"契合，模型易于用 offset 续读；B 对 token 预算更精确但分页语义模糊。

Winner: A——并辅以单行字节上限（2000）防超长行撑爆输出，兼顾两者。
