Implementation Notes

Issue #160 — 分发 prompt/command 到 commands 目录 — 2026-07-20

Design Decisions

Decision 优先 commands/，回退包根 *.md

Ambiguity: pi prompt 包的 .md 模板放在哪未统一——有的在 commands/ 子目录，有的散在包根。

Choice: 先找 <pkgDir>/commands/*.md，不存在则回退包根 *.md；回退时跳过 README.md。

Rationale: 兼容两种真实布局；跳过 README 避免把说明文档误当命令模板。

Decision 扁平化复制到 $PIGO_HOME/commands/

Ambiguity: 目标加载器 LoadUserCommandsDir 是非递归扫描。

Choice: 只复制 .md 文件本身（扁平），不保留子目录层级。

Rationale: 命令名取自文件名（去扩展名），非递归加载器要求文件直接位于 commands 目录下才能被发现。

Deviations

Deviation None

实现与 spec 一致：把 prompt 包的命令模板放到 pigo 加载的 commands 目录。无偏离。

Tradeoffs

Tradeoff 命令文件名冲突处理

Alternatives: (A) 直接以原文件名落地，可能与已有命令或别的包重名而覆盖；(B) 加包名前缀命名空间隔离。

Pros/Cons: A 保留作者原意的命令名（/review 就是 /review），符合用户预期；B 避免冲突但命令名变丑（/pi-prompts-review）。

Winner: A（本期）。命令名保真优先；冲突由 lockfile 记录的文件清单在 uninstall 时精确清理，重名覆盖的风险留作 open question。
