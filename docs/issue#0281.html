Implementation Notes

Issue #281 — /btw: 裸 /btw 重开最近侧线程 — 2026-07-24

Design Decisions

Decision 进程级状态存于 replDeps.lastBtw + lastBtwBase

Ambiguity: Issue 要求"维护进程级 lastBtw *agentcore.AgentContext"，但未规定放在哪个结构、如何区分"背景拷贝"与"侧线程问答"。

Choice: 在 replDeps 上加 lastBtw（最近侧线程）与 lastBtwBase（拷贝自主上下文的前缀长度）。REPL 循环以 &deps 调用 runBtw，故跨轮次持久，重启进程即归零。

Rationale: replDeps 已是循环持有的可变状态载体；用一个 base 下标即可在重开时只回放 Messages[base:]（侧问答），不重复打印整段主对话。

Decision 重开只回放 user/assistant 文本，省略工具活动

Ambiguity: "展示历史问答"未规定回放粒度。

Choice: replaySideHistory 只打印 UserMessage 文本（you: 前缀）与 AssistantMessage 文本，跳过 tool call / tool result。

Rationale: 侧线程重开的目的是"浏览earlier answers"，紧凑回放更可读；工具细节对回顾问答无价值。

Deviations

Deviation 会话切换清空 lastBtw 覆盖 /fork、/clone、/import，未含 /tree

Spec: AC 写"会话切换（/fork/clone/import）时清空 lastBtw"。

Implemented: 在 runForkClone 与 runImport 成功切换后 deps.lastBtw = nil；/tree 不清空。

Why: /tree 只在同一会话内移动 leaf，不是切换到另一段对话，侧线程背景仍属同一 conversation，无需失效。这与 AC 列举的三个命令一致，未偏离意图。

Tradeoffs

Tradeoff 重开沿用旧背景快照，不刷新主上下文

Alternatives: (A) 重开时保留侧线程创建时拷贝的主上下文背景；(B) 重开时用当前最新主上下文重新 seed 背景，再接上旧侧问答。

Chosen: A。Pros: 侧线程语义稳定——它是"当时那条支线"，问答与其背景自洽；实现最简。Cons: 若主对话在两次 /btw 间大幅推进，重开的侧线程看到的是旧背景。对"快速回顾侧问答"场景可接受；需要最新背景时用户可起新 /btw <q>。
