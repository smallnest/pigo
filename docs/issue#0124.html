Implementation Notes

Issue #124 — /export 与 /import 会话 (US-008) — 2026-07-17

Design Decisions

Decision JSONL export reuses the exact on-disk session schema

Ambiguity: The spec required a lossless round-trip but did not prescribe the export wire format.

Choice: WriteJSONL delegates to the same writeSessionEntries the store uses to persist a session (header line 1, one Entry per line), and ReadJSONL delegates to readSession (with the same v1/v2 migration LoadEntries applies).

Rationale: A WriteJSONL → ReadJSONL round-trip is lossless by construction — entry ids and parentIds survive verbatim, so the tree (and thus PathToLeaf/resume) behaves identically. No parallel serializer to drift out of sync.

Decision Import materializes a fresh, independent session with lineage

Ambiguity: "resume 为新会话" — but how should the imported session relate to the source?

Choice: Import assigns a fresh NewID(now), copies Model/Provider/SystemPrompt, records the source id as ParentSession, and writes the entries verbatim (ids/parentIds preserved) via SaveEntries.

Rationale: Mirrors the Fork primitive's lineage model (#122). The import is a new, independently resumable file that never touches the original; the parent link keeps provenance queryable.

Decision Self-contained HTML with inline CSS only, everything escaped

Ambiguity: "样式内联，不依赖外部网络资源" left the exact rendering open.

Choice: WriteHTML emits one <style> block, no <script>, no <link>, no external fonts or URLs; every piece of session-derived text passes through html.EscapeString before reaching the document.

Rationale: The file renders identically offline and cannot phone home. Escaping is XSS defense against a hostile session (a crafted </script> or markup cannot break out).

Decision /export and /import intercepted in the REPL loop, not as slash Actions

Ambiguity: pigo has two command mechanisms — Action closures (string→string) and REPL interceptors.

Choice: Both are intercepted directly in runREPL (like /fork//tree). /export must first persistTurn so the export reflects unsaved messages; /import must swap deps.header/agentCtx/curLeaf/persisted in place. Both are still registered as no-op builtins so /help lists them.

Rationale: A pure string→string Action closure cannot persist state or swap the live session — the same reason /fork//clone//tree are intercepted.

Deviations

None — implementation followed the spec as written. All acceptance criteria (default JSONL export, HTML export with role/tool-call highlighting, lossless round-trip, inline self-contained styles, passing session tests) are met.

Tradeoffs

Tradeoff Token-boundary guard duplicated at interceptor and in pathCommandArg

Alternatives: (a) guard only in the REPL interceptor; (b) guard only in pathCommandArg; (c) guard in both.

Chosen: Both. The interceptor uses line == "/export" || HasPrefix(line, "/export ") to decide whether to handle the line at all (so /exporter falls through to slash resolution and reports "unknown command"); pathCommandArg repeats the check defensively when extracting the arg.

Why it won: The interceptor guard is the functional one (routing); the helper guard makes pathCommandArg safe to call in isolation and self-documenting. Minor redundancy, no behavioral risk. Verified by TestREPLImportTokenBoundary.

Tradeoff HTML/JSONL selection by file extension

Alternatives: (a) an explicit --format flag; (b) infer from the output path extension.

Chosen: Extension-based (.html/.htm → HTML, everything else → JSONL), matching pi's ergonomics and the acceptance criteria's implied /export path.html syntax.

Why it won: Zero extra syntax for the common case; /export with no path defaults to <id>.jsonl. The cost is that a mistyped extension silently picks a format — acceptable for a REPL convenience command.

Tradeoff /import does not persist the current turn before switching away

Alternatives: (a) persist the outgoing session first (as /fork does); (b) just switch.

Chosen: Just switch. The REPL already calls persistTurn at the end of every turn, so by the time /import runs there are no unsaved messages to lose; the outgoing session file is already current on disk.

Why it won: Avoids an unnecessary write. /import is abandoning the current live view, not branching from it, so the /fork-style pre-persist is not needed.

Open Questions

Question Should HTML export be importable too?

Currently /import of an HTML file fails (it isn't valid JSONL) — verified by TestImportRejectsHTML. This is intentional (HTML is a read-only share format), but confirm that a clear error is the desired behavior rather than, say, a friendlier "HTML exports cannot be imported" message.
