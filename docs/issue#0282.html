Implementation Notes

Issue #282 — /btw: 独立的 model / thinking 覆盖配置 — 2026-07-24

Design Decisions

Decision 配置文件路径复用 configDir()，落在 $PIGO_HOME/btw.json

Ambiguity: Issue 要求"对标 pi-btw.json 的覆盖配置"，但未规定文件名与目录。

Choice: btwConfigPath() 复用既有 configDir()（$PIGO_HOME 或 ~/.pigo），文件名 btw.json；configDir() 返回空时视为"无配置"。

Rationale: 与 pigo 其它配置同目录，用户心智一致；命名对齐 pi-btw；空目录静默降级避免在无 HOME 环境崩溃。

Decision 每次 /btw 调用重新读取配置

Ambiguity: 未规定配置的读取时机（启动时一次 vs 每次调用）。

Choice: runBtw 开头调用 resolveBtwSettings(out, deps)，每次 /btw 都重读 btw.json。

Rationale: 编辑配置后无需重启 pigo，下次 /btw 即生效；配置文件极小，重读成本可忽略。

Decision 逐字段降级（partial config）

Ambiguity: "部分配置"如何处理未规定。

Choice: btwRunSettings 先以 deps.live 会话默认填满，再逐字段用 btw.json 覆盖；只设 thinkingLevel 时 model 仍继承会话，反之亦然。

Rationale: 用户可只覆盖关心的一项；空字段/缺字段一律"继承默认"，语义直观。

Deviations

Deviation 无效值只警告一行并降级，从不使 /btw 失败

Spec: Issue 要求"覆盖 model/thinking"，未明确错误处理策略。

Implemented: 畸形 JSON → 一行 invalid btw.json 警告后整体继承；非法 thinkingLevel → 一行警告后保留会话值；model 无法 resolve/鉴权 → 一行警告后回退会话 model+provider。全部非致命。

Why: 侧线程是"随手一问"的辅助功能，配置错误不应阻断提问；一行 dim 警告足以提示用户而不打断心流。

Tradeoffs

Tradeoff 通过 btwRunSettings 传参，而非临时改写 deps.live

Alternatives: (A) 把解析后的 model/thinking 打包成 btwRunSettings，穿过 askSide/btwFollowUpLoop 显式传递；(B) 进入 /btw 时临时改 deps.live，退出时还原。

Chosen: A。Pros: deps.live 永不被侧线程触碰，覆盖绝无可能泄漏进主会话（FR-8）；无需 defer 还原、无 panic 泄漏风险；可测（TestBtwConfigDoesNotMutateSession）。Cons: 三个函数签名各加一个 settings 参数，调用点略啰嗦。

Rationale: 隔离正确性 > 签名简洁；B 的"改了再还原"在 Ctrl+C/异常路径下极易漏还原，风险不值当。

Tradeoff model 覆盖复用 resolveProvider 而非新解析逻辑

Alternatives: (A) 复用 /model 用的 resolveProvider(model, baseURL, protocol, "")；(B) 为 btw 写独立的 provider 解析。

Chosen: A。Pros: 与 /model 行为完全一致（preset 目录、ollama 前缀、协议推断），零重复。Cons: 沿用会话的 baseURL/protocol，btw.json 无法单独覆盖 base URL —— 但本 issue 只要求 model/thinking，未超范围。

Open Questions

Question btw.json 是否需要支持覆盖 baseURL / protocol / provider

Assumption: 本 issue 仅覆盖 model 与 thinkingLevel；model 的 provider 由 resolveProvider 从会话的 baseURL/protocol 推断。

Verify: 确认"侧线程用另一 provider/自定义 base URL"不在需求内。若将来需要跨 provider 的侧线程（如主用 anthropic、侧用本地 ollama），需扩展 btwConfig 增加 baseURL/protocol 字段。
