Design Decisions

Decision ResolveThinking 返回 (string, bool) 表达三态

Ambiguity: 验收要"nil value=关闭该档、缺 key=不支持"，但未规定解析函数签名如何区分。

Choice: (wire, true)=支持有值；("", true)=支持但禁用（nil value）；("", false)=不支持（缺 key 或 nil map）。

Rationale: bool 区分"支持/不支持"，空串区分"有值/禁用"，正好还原 map[level]*string 的三态而不暴露指针给调用方。

Decision clampReasoning 找"≤ 请求档的最高支持档"

Ambiguity: 验收只举 xhigh→high 一例，未定义完整降级规则。

Choice: 按 thinkingOrder 从请求档向下找 map 中存在的最高档；找不到返回 not-found。

Rationale: 泛化"xhigh→high"为通用降级——任意不支持档都退到其下最近支持档，覆盖模型能力表参差的情况。
