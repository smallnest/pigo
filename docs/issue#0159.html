Implementation Notes

Issue #159 — 分发 skill 到 skills 目录 — 2026-07-20

Design Decisions

Decision 复用嵌套布局 <skillsDir>/<name>/SKILL.md

Ambiguity: Issue 只说"分发到 skills 目录"，未规定单文件还是目录布局。

Choice: 整包树复制到 SkillsDir()/<name>/，正好命中 runtime.LoadSkillsDir 已支持的嵌套布局（每个 skill 一个目录，内含 SKILL.md 及其支撑文件）。

Rationale: 无需改动 skill 加载器；npm skill 包本身就是"SKILL.md + 支撑文件"的 bundle，目录布局是天然匹配，且能容纳 references/ 等附属资源。

Deviations

Deviation None

实现与 spec 一致：读取分类结果为 skill 的包，复制到 skills 目录使其可被发现。无偏离。

Tradeoffs

Tradeoff skills 目录不在 $PIGO_HOME 下

Alternatives: (A) 装到 $PIGO_HOME/skills；(B) 装到 pigo 实际加载的 SkillsDir()（~/.agents/skills 或 PIGO_SKILLS_DIR）。

Pros/Cons: A 让所有 pi 包集中在 PIGO_HOME 下，整齐；但 pigo 运行时根本不从那里加载 skill，装了也不生效。

Winner: B。正确性优先——必须装到运行时真正扫描的目录。#154 的 SkillsDir() 已把这个例外固化下来。

Open Questions

Question skill 目录命名用包名还是 frontmatter name

Assumption: 用 npm 包名作为 <name>/ 目录名。
