Design Decisions

Decision Use JohannesKaufmann/html-to-markdown/v2 for HTML→Markdown

Ambiguity: The spec says "HTML→markdown 简化" but does not mandate fidelity or a specific library.

Choice: Wrap the html-to-markdown/v2 library (base + commonmark plugins). The base plugin strips head/script/style/iframe/input by default; we additionally register nav/footer/header/aside/svg/form/template for removal via Register.TagType(tag, TagTypeRemove, ...). The shared converter is built once with sync.OnceValue.

Rationale: Initially this was a hand-rolled ~190-line reducer over x/net/html to avoid a dependency. On review that judgment was reversed: the library adds only 3 small transitive deps (the library, its dom helper, and cascadia; x/net and x/text were already present) and gives proper tables, nested lists, commonmark-conformant escaping, and community-maintained edge cases — versus re-implementing a tested wheel. The library's Converter is documented and race-tested for concurrent ConvertString, so one shared instance is safe.

Decision Cross-origin redirect reported as a normal result, not an error

Ambiguity: "跨域重定向返回调用方而非自动跟随" — return to caller, but as an error or as content?

Choice: A blocked redirect returns a non-error AgentToolResult whose text names the target URL and invites an explicit re-fetch, with Details.redirect set.

Rationale: This is not a failure — it is a safety decision the model should act on. Framing it as content (not an error) lets the model read the target and decide, which is the point of "return to caller".

Decision prompt is echoed, not executed

Ambiguity: "输入 URL + 可选 prompt" — does the tool summarize per the prompt?

Choice: The prompt is echoed into the result framing ("Prompt: ..."); the tool does not itself call a model.

Rationale: Summarization is the agent loop's job — the tool returns raw material plus the stated intent, keeping the tool deterministic and testable without a model dependency.

Deviations

None — implementation followed the spec as written.

Tradeoffs

Tradeoff Library vs. hand-rolled HTML→Markdown reducer

Alternatives: (a) hand-rolled reducer over x/net/html (no new dep); (b) the html-to-markdown/v2 library.

Chosen: (b). The first cut shipped (a) to avoid a dependency, but a follow-up reversed it after measuring the actual cost: only 3 small transitive packages, none heavy.

Why: The hand-rolled version needed custom edge-case patches (e.g. collapseInlineSpace to keep spacing around inline links — a bug caught in review) and still lacked tables, nested lists, and proper escaping. Delegating to a maintained, race-tested library removes that whole bug class for a negligible dependency footprint.

Tradeoff HTTP→HTTPS upgrade vs. reject plain HTTP

Alternatives: (a) reject http:// outright; (b) silently upgrade to https://.

Chosen: (b) upgrade in normalizeFetchURL, exactly as the acceptance criterion states ("HTTP 升级为 HTTPS").

Why: Upgrading is friendlier (most sites serve HTTPS) and matches the spec verbatim. A site that only serves plain HTTP will fail the request, surfaced as a structured error — acceptable given the security intent.

Tradeoff Redirect blocking via CheckRedirect vs. manual hop loop

Alternatives: (a) set Client.CheckRedirect to reject cross-origin hops; (b) disable redirects and follow them manually with per-hop origin checks.

Chosen: (a) — same-host/same-scheme redirects are followed, cross-origin ones stop with a typed errRedirectBlocked that errors.As unwraps from the *url.Error.

Why: CheckRedirect is the idiomatic hook; it preserves in-origin redirects (common, harmless) while blocking cross-origin ones. The manual loop would re-implement stdlib redirect handling for no added safety.

Open Questions

Question Are the size/time caps right for typical use?

Defaults: 30s timeout, 5 MiB body read, 100 KiB rendered Markdown. These bound context and memory but may truncate long articles. Verify the Markdown cap is generous enough for the intended pages, or whether it should be configurable per call.

Question Should webfetch be gated behind project trust?

The tool makes outbound network requests. Issue #134 (Project Trust) may want to gate network-capable tools. Confirm whether webfetch should be disabled in untrusted projects once #134 lands.
