Issue #235 — Wire model-name inference into resolveProvider — 2026-07-23

Design Decisions

Decision Inference is skipped when --base-url is set

Ambiguity: The issue says inference fires only when --provider, --protocol, and --base-url are all unset, but resolveProvider reaches step 4 with protocol=="" and provider=="" already guaranteed — only base URL still needs a guard.

Choice: Gate the inference branch on strings.TrimSpace(baseURL) == "".

Rationale: A user-supplied base URL signals a custom/self-hosted endpoint; inferring a first-party provider (with its own default base URL) would override the user's endpoint. Skipping keeps the OpenRouter default, which respects the given base URL.

Decision Delegate hits to resolveNamedProvider

Ambiguity: The issue says "delegate to resolveNamedProvider" but that function also accepts a protocol arg.

Choice: Pass the (empty) protocol through unchanged.

Rationale: Reuses the exact registry-driven path (protocol, base-URL precedence, <PROVIDER>_API_KEY env) that explicit --provider uses, so inferred and explicit selection are identical downstream. An empty protocol never conflicts with the spec's own protocol.

Tradeoffs

Tradeoff Placement after preset/prefix, before default

Alternatives: (a) infer early, before the preset catalog; (b) infer late, just before the OpenRouter fallback.

Chosen: (b).

Why it won: The preset catalog and ollama//nvidia/ prefixes are more specific, curated signals; letting them win first preserves all existing behavior, and name inference only rescues ids that would otherwise have silently defaulted to OpenRouter.
