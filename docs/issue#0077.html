Issue #77 — (4/6) 抽出 runtime 子包并删除旧包 — 2026-07-10

Design Decisions

Decision AST-based requalification instead of textual sed

Ambiguity: The moved files reference ~157 moved symbols by bare name (they compiled via shims). The spec says "correctly export" but doesn't prescribe how to rewrite the now-dangling bare references to their real home packages.

Choice: An AST-based rewriter that skips selector tails, struct-literal field keys, and locally-declared names, qualifying only true package-level identifier uses.

Rationale: A textual replace would corrupt field keys like Model:, Content:, Provider: in struct literals and method names. The AST walk distinguishes "use of an imported symbol" from "a field/label that happens to share the name" — the only safe way to touch 157 symbols across 8 files.

Decision PrintMode / StreamJSONMode re-exported as const, not var

Ambiguity: The shim must re-export these two. Whether to bind them as var (like functions) or re-declare as const depends on their kind.

Choice: const PrintMode = runtime.PrintMode — they are typed HeadlessMode iota consts.

Rationale: A var binding would demote a compile-time constant to a runtime value, breaking any use in a const expression or switch-case label. Const re-declaration preserves their constant-ness and type identity through the alias.

Deviations

Deviation internal/agent NOT deleted — reduced to shim-only

Spec: US-004's AC states "原 internal/agent 包被完全清空并删除" (fully emptied and deleted).

Implemented: All production/loop files moved out; internal/agent now contains ONLY the four re-export shims (agentcore_shim.go, provider_shim.go, agenttool_shim.go, runtime_shim.go). The package still exists.

Why: External callers (cmd/, internal/tui, internal/session) still reference ~49 agent.* symbols; migrating them is explicitly US-005/#78's job. Deleting internal/agent now would break go build ./... — and every US's hard gate is a green build. The deletion happens in #78 once the last caller is migrated. This preserves the incremental one-layer-per-US verification guarantee.

Tradeoffs

Tradeoff Delete agent now (+migrate callers) vs defer to #78

Alternatives: (a) fold #78's caller migration into #77 so agent can be deleted as the AC demands; (b) keep #77 scoped to the runtime extraction, leave agent as a shim, delete in #78.

Chosen: (b).

Why it won: (a) would merge two issues, doubling the blast radius and breaking the PRD's bottom-up plan where each US is a compilable checkpoint. (b) keeps #77 reviewable in isolation and the build green at every step; the AC's "delete" clause is simply satisfied one issue later than literally worded.
