Implementation Notes

Issue #125 — /copy 与 /session 命令 (US-009) — 2026-07-17

Design Decisions

Decision Clipboard via shell-out, no cgo / third-party dependency

Ambiguity: The spec named pbcopy/xclip/wl-copy but did not mandate an implementation strategy.

Choice: A new internal/clipboard package probes for platform utilities (pbcopy on darwin; wl-copy → xclip → xsel on linux; clip on windows) via exec.LookPath and pipes the payload to the first found.

Rationale: Keeps the dependency graph pure-stdlib (no cgo, no atotto/clipboard), matching the project's zero-external-dep posture. ErrUnavailable lets the caller degrade cleanly.

Decision /copy targets the most recent non-empty assistant text

Ambiguity: "最近一条 assistant 文本" — assistant messages can be tool-call-only with no text.

Choice: Walk the message list backwards, skipping assistant messages whose text (via ContentToText) is empty, and copy the first with real text.

Rationale: A tool-call-only turn has nothing meaningful to copy; the user means the last thing the agent actually said.

Decision /session stats derived from the in-memory context

Ambiguity: Stats could come from the on-disk file or the live context.

Choice: Message count, token estimate (compaction.EstimateContextTokens), and compaction count are computed from deps.agentCtx.Messages; id/created/model come from the header + live run config.

Rationale: The live context is the source of truth for the current turn and includes not-yet-persisted messages, so the numbers match what the user sees.

Deviations

None — implementation followed the spec. /copy degrades to printing when no utility is present (explicitly required); /session shows session name, message count, tokens, model, creation time, and compaction count as specified.

Tradeoffs

Tradeoff Clipboard test cannot assert the real OS clipboard changed

Alternatives: (a) integration test against the real clipboard; (b) contract test with a fake utility on PATH.

Chosen: Contract test — empty PATH → ErrUnavailable; a planted shell shim named after the platform candidate → success. The REPL-side degrade path is covered by TestREPLCopyDegradesToPrint.

Why it won: A real-clipboard test is non-hermetic (would clobber the developer's clipboard and fail in CI without a display server). The contract is exactly what the REPL relies on.

Tradeoff /session token count is an estimate, not exact

Alternatives: (a) call the provider's tokenizer; (b) reuse the existing heuristic estimator.

Chosen: Reuse EstimateContextTokens (prefers the last real usage block, estimates the trailing tail) — the same number /compact reports.

Why it won: Consistency with /compact and zero extra network round-trips; the label "tokens (est)" makes the approximation explicit.

Open Questions

Question Should /copy support copying a specific message (e.g. /copy N)?

Current /copy always takes the latest assistant reply. pi's behavior is the same, so I kept it minimal, but a "/copy N" selector (like /fork N) could be a small follow-up if users want to grab an earlier reply. Confirm whether the latest-only behavior is sufficient.
