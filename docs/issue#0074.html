Design Decisions

Decision Single alias shim file instead of scattered aliases

Ambiguity: The PRD's transition strategy said "旧 internal/agent 可临时用类型别名 type X = agentcore.X 保持编译" but didn't prescribe where the aliases live.

Choice: One dedicated internal/agent/agentcore_shim.go collects every re-export — type aliases, const re-declarations, constructor var bindings — with a header comment marking it as transitional (removed in later steps).

Rationale: Keeps the 4 unmoved files (providers/headless/stream_response/tool_executor) untouched except for the symbol cut-outs, and makes the shim trivially deletable in US-004/005 when call sites migrate to reference agentcore directly.

Decision Export helpers as ContentToText / LastAssistantOf / EmitFunc

Ambiguity: The three hoisted symbols (contentToText, lastAssistantOf, emitFunc) are unexported, but once in agentcore they must be reachable across the package boundary.

Choice: Capitalize them in agentcore (ContentToText, LastAssistantOf, EmitFunc) and alias the lowercase names back in the shim (var contentToText = agentcore.ContentToText, type emitFunc = agentcore.EmitFunc).

Rationale: Cross-package visibility requires export; the lowercase aliases keep every existing call site in package agent compiling verbatim. Directly satisfies PRD 陷阱 2 (emitFunc hoist) and the shared-helper hoist.

Decision Generic EventStream via alias + thin wrapper

Ambiguity: Go generic functions cannot be bound to a var, unlike ordinary functions.

Choice: type EventStream[T any, R any] = agentcore.EventStream[T, R] (generic type alias, Go 1.24+) plus a hand-written wrapper func NewEventStream[T any, R any](buffer int) *agentcore.EventStream[T, R].

Rationale: Constructors like NewTextContent alias cleanly as var, but the generic NewEventStream needs an explicit wrapper — the only forwarder that isn't a one-liner var.

Deviations

Deviation Hook types placed in helpers.go, not a hooks.go

Spec: US-001 lists hooks.go among the migrated files and says the hook func types should "统一收敛到 agentcore" without naming the target file.

Implemented: AfterToolCallResult moved with hooks.go (it already lived there), but the func types cut from tool_executor.go (PrepareArgumentsFunc, BeforeToolCallDecision, BeforeToolCallFunc, AfterToolCallFunc) landed in a new agentcore/helpers.go alongside the relocated helpers.

Why: These types originated in tool_executor.go, not hooks.go; grouping them with the other cross-layer relocations keeps hooks.go a verbatim git mv (clean diff) and co-locates all "hoisted from elsewhere" symbols in one reviewable file.

Tradeoffs

Tradeoff git mv + rename vs copy-paste

Alternatives: (a) git mv each leaf file then flip the package clause; (b) create fresh files in agentcore and delete the originals.

Chosen: (a). Pros: git tracks the move, so history/blame follow the code and the diff shows a 1-line package-clause change per file. Cons: none material.

Why it won: Preserves authorship history across the split — valuable for a 64-file refactor where later archaeology on "why is this type shaped this way" should still resolve.

Tradeoff Keep shim in package agent vs migrate callers now

Alternatives: (a) alias shim so package agent compiles unchanged this step; (b) update every intra-agent call site to reference agentcore. immediately.

Chosen: (a), per the PRD's bottom-up, one-layer-per-US plan.

Why it won: Bounds each US to a compilable, testable checkpoint. Migrating callers is explicitly US-005's job; doing it now would balloon #74's blast radius and break the incremental verification guarantee.
