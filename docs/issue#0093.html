Implementation Notes

Issue #93 — TUI: 流式状态指示与工具调用卡片 — 2026-07-10

Design Decisions

Decision spinner tick 只在 running 时自续，靠 tea.Batch 与 drain 一起启动

Ambiguity: AC 要求「tick 命令驱动、不阻塞 Update、run 结束后停止动画」，未规定如何停。

Choice: startRun 返回 tea.Batch(drain, m.spinner.Tick)；Update 收到 spinner.TickMsg 时若 !running 直接返回 nil（不再续 tick），否则 m.spinner.Update 返回下一个 tick。

Rationale: spinner 的自续机制是"处理 tick → 返回下一个 tick"，只要在 idle 时掐断续命，动画自然在 run 结束后一帧内停下，无需显式 Stop；tick 走 bubbletea 命令队列，不阻塞 Update。

Decision 工具卡片 = 复用 #89 主题样式 + #91 折行后再上色

Ambiguity: AC 要求 tool-call/tool-result「以带样式的卡片呈现，区别于普通文本」，未规定卡片具体形态。

Choice: 新增 styleEntry(kind, text)，按 entry 种类套用 theme.toolCall/theme.toolResult 等样式；在 wrapWidth 折行「之后」再上色。tool-call 用 magenta 头 + ⚙ 前缀，tool-result 用 grey 体 + ↳ 前缀，与 assistant/user 配色区分。

Rationale: 主题层（#89）已备好每类样式，直接复用避免重复定义配色；先折行后上色保证 #91 的显示宽度计算落在裸 rune 上而非 ANSI 转义（否则宽度算错）——正是 #91 open question 预警的顺序。

Decision spinner 选用 MiniDot，样式取主题 accent 色

Ambiguity: AC 未指定 spinner 具体样式。

Choice: spinner.New(WithSpinner(MiniDot), WithStyle(theme.accent))，运行提示语整体也套 accent。

Rationale: MiniDot 单宽 braille 帧稳定不抖动、对中文终端友好；accent 色与主题统一。

Deviations

None 实现遵循 spec

AC 五项（spinner 运行动画 + 提示、工具卡片区别于普通文本、保留 steering/Ctrl+C 提示语义、tick 驱动不阻塞且结束停止、build+test 全绿）均按字面满足。

Tradeoffs

Tradeoff 卡片=着色行，而非带边框的多行 box

Alternatives: (a) 用 lipgloss 边框/padding 画真正的多行卡片框；(b) 仅用配色 + 前缀 glyph 区分的"轻卡片"。

Chosen: (b)。

Why it won: 多行边框框在 viewport 折行、CJK 宽字符对齐下极易错位，且会与 #91 的宽度计算、#94 的 markdown 排版打架；配色+glyph 已满足"视觉区别于普通文本"的 AC，且 NO_COLOR 下退化为 glyph 仍可区分。真正的边框 box 留待样式体系稳定后（#94/#95 之后）再考虑。

Tradeoff theme 固定为 dark，未接入运行时切换

Alternatives: (a) Model 固定 buildTheme(themeDark)；(b) 本 issue 就加 light/dark 切换入口。

Chosen: (a)。

Why it won: 主题切换是 #89 的范畴，本 issue 只消费主题样式；NO_COLOR 已由 buildTheme 处理。切换 UI 留给后续。

Open Questions

Question spinner 帧宽与 viewport 底部行

运行提示当前渲染在 viewport 之外的固定底部区（composer 上方）。若后续把运行状态并入可滚动区域，需复核 spinner 动态帧宽对 #91 宽度对齐的影响（MiniDot 单宽已规避）。

Question 是否需要区分「工具执行中」与「工具已完成」的卡片态

当前 tool-call 与 tool-result 是两条独立 transcript entry。zero 的 tool card 会把 call→result 合并为一张带状态的卡片。是否需要该合并（含 pending/done 图标切换）留待确认；本 issue 先按 PRD 的两态着色实现。
