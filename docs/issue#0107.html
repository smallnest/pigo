Issue #107 — REPL 斜杠命令、技能与模型切换 — 2026-07-13

Design Decisions

Decision 斜杠输入统一经 ResolveOutcome 分流

Ambiguity: Spec 要求 action / prompt / 技能 / 未知命令四种分支各有不同行为，但未规定分流机制。

Choice: REPL 对以 / 开头的输入统一调用 SlashRegistry.ResolveOutcome，返回 SlashOutcome{Kind, Message, Prompt}：SlashAction 打印 Message 不发起 run；prompt/技能返回展开的 Prompt 作为本轮输入；未知命令返回 error 打印 unknown command "/foo"。

Rationale: 单一分流点让 REPL 循环保持极薄，四种行为的差异完全收敛在 registry 一侧，是既有 seam 的自然复用。

Decision /model 切换通过 liveRunConfig 生效于下一轮

Ambiguity: "下一轮生效" 的实现方式未规定。

Choice: registerLiveCommands 把 /model 的闭包绑定到共享的 *liveRunConfig；命令经 resolveProvider 解析后原地改写 live 的 model/provider，streamRun 每轮从 live 读取，因此切换在下一轮自然生效。

Rationale: live 是 REPL 主 goroutine 上唯一可变的运行配置，无需锁；action 命令与 run 都同步串行执行。

Deviations

None — 斜杠命令分流、技能加载（~/.agents/skills 经 buildSlashRegistry）、/model//models//help 均在 #106 落地时已实现并覆盖测试。本 Issue 仅补齐 US-005/US-007 的 /model 运行中切换验收测试，未偏离 spec。

Tradeoffs

Tradeoff 切换到 Ollama preset 作为测试目标 vs mock provider 解析

Alternatives: (A) 测试中 /model 切到 ollama/llama3.3（Ollama 预置、无需 API key）；(B) 注入一个假的 resolveProvider。

Pros/Cons: (A) 走真实 resolveProvider 路径，验证的是生产逻辑，且 Ollama 无需密钥不触发早退错误；(B) 更隔离但绕过了真实解析，测试价值低。

Why chosen: 选 (A)——验收标准要求 /model <id> "通过 resolveProvider 切换"，测试应打到真实路径。
