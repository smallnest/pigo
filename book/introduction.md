# 引言：从一行命令说起

你在终端里敲下一句话，回车，几秒钟后代码被读懂、文件被修改、命令被执行——这背后不是魔法，而是一套可以被拆开、看清、复现的机制。本书要做的，就是把这套机制彻底拆解，再用 Go 语言从零把它重新装配起来。

我们的解剖对象是 `pi`，一个短小精悍的命令行 AI Agent。而我们手中的手术刀，是它的 Go 复刻版本 **pigo**（github.com/smallnest/pigo）。全书将像庖丁解牛那样，顺着源码的纹理，一刀一刀把一个真实可用的 Agent 剖开给你看。

## Agent 到底是什么

抛开营销话术，一个命令行 AI Agent 的本质可以写成一个朴素的公式：

> **Agent = LLM + 上下文（context） + 工具（tools）**

- **LLM** 是大脑。它负责理解你的意图、做出判断、决定下一步做什么。但它本身既看不到你的文件，也执行不了任何命令——它只会输出文本。
- **上下文（context）** 是记忆与视野。系统提示、历史对话、文件内容、工具返回结果，都要被组织进有限的上下文窗口里，喂给 LLM。上下文如何构造、如何压缩、如何持久化，直接决定 Agent 的"聪明"与"健忘"。
- **工具（tools）** 是手脚。读文件、写文件、执行 shell、搜索代码……LLM 通过结构化的工具调用把意图落到实处，再把执行结果作为新的上下文回传。

把三者用一个循环串起来——LLM 思考 → 调用工具 → 观察结果 → 再思考——就得到了 Agent 的核心运行时。整本书的所有章节，本质上都是在回答一个问题：**这三样东西，以及串联它们的循环，在工程上究竟如何实现？**

## 为什么是 pigo：pi 的 Go 复刻

`pi` 用脚本语言写成，胜在直观。而 **pigo 是 pi 的一次 Go 复刻（reimplementation）**：在保持行为对齐的前提下，用 Go 的类型系统、并发模型和清晰的包边界，把每个模块重写一遍。

选择"复刻"而非"发明"，是本书刻意的教学策略：

- **有参照系**：每一处实现都能对照原版 `pi` 的行为，判断对错有据可依，而不是凭空设计。
- **看得见取舍**：从动态语言迁移到静态类型的 Go，接口怎么抽象、错误怎么传递、并发怎么组织，这些工程决策会被放到台面上讨论。
- **可运行、可修改**：pigo 是一个真实、完整、能跑起来的项目，而非教学玩具。你读完每一章，都能回到仓库里亲手验证、改写、扩展。

## 本书的读法：对照源码与 commit，逐模块「庖丁解牛」

这不是一本讲概念的书，而是一本"拿着源码逐行拆解"的书。建议你这样读：

1. **对照源码读**：每一章都锚定 pigo 仓库中的具体模块与文件。读到关键实现时，请打开对应源码同步查看——文字负责讲清"为什么这样写"，源码负责呈现"到底怎么写"。
2. **顺着 commit 读**：一个功能往往不是一步到位的。我们会借助提交历史（commit），还原某个模块从雏形到成型的演进过程，让你看到设计是如何被逐步逼近的。
3. **逐模块下刀**：全书按 Agent 的自然结构切分——从最外层的 CLI 装配，到核心的 Agent 循环，再到 Provider 层、工具系统、上下文管理，直至安全与生态。每一"刀"都落在结构的关节处，拆开一个自洽的模块，看清它的输入、输出与边界，再拼回整体。

换句话说：**文字 + 源码 + commit** 三者对照，才是本书的完整阅读姿势。只读文字，你会觉得"好像懂了"；配合源码与提交历史，你才能真正"庖丁解牛"，游刃有余。

## 读前须知：前置知识

为了顺畅地跟上拆解节奏，建议你具备以下基础：

- **Go 语言**：能读写 Go 代码，理解 struct、interface、goroutine 与 channel、error 处理惯例，以及 Go Modules 的基本用法。你不需要是专家，但要能读懂中等复杂度的 Go 工程。
- **LLM API 基础**：了解与大模型交互的基本概念——system/user/assistant 消息、token 与上下文窗口、流式输出（streaming），以及"函数调用 / 工具调用（function calling / tool use）"的请求与响应格式。知道一次 Chat Completions 调用长什么样即可。
- **命令行与 Git**：熟悉终端操作，会用 `git log`、`git diff` 查看提交历史——这是"顺着 commit 读"的前提。

如果对上述某一项还不熟悉，也不必却步：正文会在关键处补充必要背景，你完全可以边读边补。

## 建议阅读路径

全书章节是按 Agent 的构造顺序编排的，**从头到尾顺序阅读**能获得最完整的体验。如果你想按需切入，这里给出一条推荐路线：

- **打地基**：先读第1章，搞清 pigo 是如何从一条命令启动、把各个组件"装配"成一个可运行程序的。
- **抓核心**：接着精读第2章的 Agent 循环——这是全书的心脏，LLM、上下文与工具正是在这里被串成一个整体。理解了它，其余章节都是对它的展开。
- **看接入与执行**：第3章讲 Provider 层，即如何统一对接不同的大模型服务；第4章讲工具系统，即 LLM 的"手脚"如何被定义、调用与约束。
- **管好记忆**：第5章的上下文压缩与第6章的会话持久化，回答"如何在有限窗口里记住足够多、并把对话存下来"。
- **守住边界**：第7章聚焦信任与安全——一个能执行命令的 Agent，如何避免闯祸。
- **再进阶**：第8章的子 Agent（sub-agent）机制，展示如何用 Agent 调度 Agent 来拆解复杂任务。
- **看全景**：最后借助讲生态与扩展的章节，把 pigo 放回更大的工具版图中审视。

如果你时间有限、只想抓住精髓，那么第1章与第2章是无论如何都不该跳过的两章：它们分别交代了"程序怎么搭起来"与"Agent 怎么转起来"。

好了，磨刀已毕。让我们从第1章开始，看 pigo 是如何把一条命令，变成一个会思考、会动手的 AI Agent 的。
